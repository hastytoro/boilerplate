Fit and Position

Understand that the img element is fundamentally weird. As it’s known as a “replacing element”. Unlike other DOM Nodes the browser replaces the img tag with a foreign entity, being the image itself. So images aren't like other elements and they have rules we need to consider:

  • Images are "go with the flow" elements.

    Meaning by default they inline and we can't give those elements height, since that would mess with layout. And yet images can be given a height.

  • Images come with their own intrinsic size.

    They based on the dimensions of the image file. Here our img has a natural size of 400×300.

  • Images also have an intrinsic aspect ratio.

    If we only supply one dimension width or height, the other dimension scales up or down as-needed -- to preserve aspect ratio.

Image also have a intrinsic aspect ratio. Meaning if we only supply one dimension, either width or height, the other dimension scales up or down as-needed, to preserve the aspect ratio.

By default, the image will distort. :(

The default behavior is almost never what we want it to do. But the better alternative in most cases is to crop the image, trimming off any excess that can't fit in the specified measurement.

Modern CSS includes tools that help like object-fit.

- object-fit: fill (default)
- object-fit: contain
- object-fit: cover (mostly used)

NOTE: object-fit property works on images, but it isn't exclusive to images. It can also be used on video tags. In general, it's less relevant with videos.

When using the cover value, the browser will crop our image so that we see the very center of the image, and trim off the edges. But in some cases, we'll want to use a different anchor point.
